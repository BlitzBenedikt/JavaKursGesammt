Zwei wichtige OOP-Konzepte: Vererbung und Polymorphismus

Was wir schon über OOP wissen

Wir haben bereits gelernt:

  • Eine Klasse ist wie ein Bauplan für Objekte
  • Eine Klasse hat Eigenschaften (was ein Objekt weiß) und Methoden (was ein Objekt tun kann)
  • Ein Objekt ist eine "lebendige" Instanz einer Klasse

Neues Konzept: Vererbung

Vererbung bedeutet, dass eine Klasse alle Eigenschaften und Methoden einer anderen Klasse übernehmen (erben) kann.

Beispiel aus dem echten Leben:

Wie in einer Familie! Kinder erben Eigenschaften von ihren Eltern:

  • Augenfarbe, Haarfarbe, Körpergröße, ...
  • Kinder haben aber auch eigene, neue Eigenschaften!

In Java sieht Vererbung so aus:

// Elternklasse (auch "Basisklasse" genannt)
class Tier {
    String name;
    
    void essen() {
        System.out.println(name + " isst.");
    }
}

// Kindklasse (erbt von Tier)
class Hund extends Tier {
    // "extends Tier" bedeutet: Hund erbt alles von Tier
    
    // Zusätzliche Methode, die nur Hunde haben
    void bellen() {
        System.out.println(name + " bellt.");
    }
}

// Verwendung:
Hund bello = new Hund();
bello.name = "Bello";    // Eigenschaft von Tier
bello.essen();           // Methode von Tier
bello.bellen();          // Eigene Methode von Hund

Wichtig zu verstehen:

  • Die Kindklasse (Hund) erbt alle Eigenschaften und Methoden der Elternklasse (Tier)
  • Die Kindklasse kann zusätzliche eigene Eigenschaften und Methoden haben
  • Wir benutzen das Schlüsselwort extends, um Vererbung in Java zu nutzen

Aufgabe 1: Einfache Vererbung üben

Erstelle eine Elternklasse Fahrzeug und eine Kindklasse Auto.

  1. Erstelle eine Klasse Fahrzeug mit:
    • Eigenschaft: name (String)
    • Methode: fahren() - gibt aus: "[name] fährt."
  2. Erstelle eine Klasse Auto, die von Fahrzeug erbt, mit:
    • Zusätzliche Methode: hupen() - gibt aus: "[name] hupt: Tuut tuut!"
  3. Teste deine Klassen in einer main-Methode
class Fahrzeug {
    String name;
    
    void fahren() {
        System.out.println(name + " fährt.");
    }
}

class Auto extends Fahrzeug {
    void hupen() {
        System.out.println(name + " hupt: Tuut tuut!");
    }
}

public class FahrzeugDemo {
    public static void main(String[] args) {
        Auto meinAuto = new Auto();
        meinAuto.name = "VW Golf";
        
        // Eine Methode von Fahrzeug aufrufen
        meinAuto.fahren();
        
        // Eine Methode von Auto aufrufen
        meinAuto.hupen();
    }
}

/* Ausgabe:
VW Golf fährt.
VW Golf hupt: Tuut tuut!
*/

Neues Konzept: Methoden überschreiben

Eine Kindklasse kann eine Methode der Elternklasse neu definieren (überschreiben).

Das bedeutet: Die Methode hat den gleichen Namen, arbeitet aber anders.

class Tier {
    String name;
    
    void machGeräusch() {
        System.out.println("Das Tier macht irgendein Geräusch.");
    }
}

class Hund extends Tier {
    @Override  // Diese Zeile ist optional, aber hilfreich
    void machGeräusch() {
        System.out.println(name + " bellt: Wau wau!");
    }
}

class Katze extends Tier {
    @Override
    void machGeräusch() {
        System.out.println(name + " miaut: Miau!");
    }
}

Merke:

  • Die Annotation @Override ist optional, hilft aber zur Fehlervermeidung
  • Die überschriebene Methode muss genau den gleichen Namen und Parameter haben
  • Wenn ein Objekt der Kindklasse die Methode aufruft, wird die neue Version verwendet

Was ist Polymorphismus?

"Polymorphismus" ist ein langes Wort und bedeutet "verschiedene Formen annehmen".

Beispiel aus dem echten Leben:

Stell dir vor, du sagst zu verschiedenen Personen: "Guten Morgen!"

  • Deine Mutter antwortet: "Guten Morgen, hast du gut geschlafen?"
  • Dein Lehrer antwortet: "Guten Morgen, bereit für den Unterricht?"
  • Dein Freund antwortet: "Morgen! Spielen wir später?"

Der gleiche Gruß - aber jeder antwortet anders!

In Java bedeutet Polymorphismus:

  1. Verschiedene Klassen können die gleiche Methode haben, aber sie arbeitet unterschiedlich
  2. Du kannst Objekte verschiedener Klassen über eine gemeinsame Elternklasse ansprechen

Der wichtigste Teil: Elternklassen-Variablen können Kindklassen-Objekte speichern!

Tier einTier = new Hund();  // Das ist erlaubt!
einTier.name = "Bello";
einTier.machGeräusch();     // Ruft die Hund-Version auf: "Bello bellt: Wau wau!"

Das ist wichtig zu verstehen: Obwohl einTier eine Variable vom Typ Tier ist, wird die machGeräusch()-Methode des Hund-Objekts aufgerufen!

Java schaut zur Laufzeit, welches Objekt wirklich in der Variable steckt, und führt dann die richtige Methode aus.

Aufgabe 2: Tier-Stimmen mit Polymorphismus

Erstelle ein Programm mit Tieren, die unterschiedliche Geräusche machen.

  1. Erstelle eine Basisklasse Tier mit:
    • Eigenschaft: name (String)
    • Methode: machGeräusch() - gibt einen allgemeinen Text aus
  2. Erstelle mindestens zwei Tierarten (z.B. Hund, Katze), die von Tier erben
  3. Jede Tierart soll die machGeräusch()-Methode überschreiben
  4. Zeige Polymorphismus mit folgenden Schritten:
    • Erstelle ein Array vom Typ Tier
    • Speichere verschiedene Tierobjekte im Array
    • Rufe machGeräusch() für jedes Tier im Array auf
class Tier {
    String name;
    
    void machGeräusch() {
        System.out.println(name + " macht ein Geräusch.");
    }
}

class Hund extends Tier {
    @Override
    void machGeräusch() {
        System.out.println(name + " bellt: Wau wau!");
    }
}

class Katze extends Tier {
    @Override
    void machGeräusch() {
        System.out.println(name + " miaut: Miau!");
    }
}

class Kuh extends Tier {
    @Override
    void machGeräusch() {
        System.out.println(name + " muht: Muuuuh!");
    }
}

public class TierDemo {
    public static void main(String[] args) {
        // Einzelne Tiere erstellen
        Hund bello = new Hund();
        bello.name = "Bello";
        
        Katze minka = new Katze();
        minka.name = "Minka";
        
        Kuh emma = new Kuh();
        emma.name = "Emma";
        
        // Normale Aufrufe ohne Polymorphismus
        System.out.println("Normale Aufrufe:");
        bello.machGeräusch();
        minka.machGeräusch();
        emma.machGeräusch();
        
        // Ein Array von Tieren erstellen - DAS ist Polymorphismus!
        System.out.println("\nMit Polymorphismus:");
        Tier[] tiere = new Tier[3];
        tiere[0] = bello;    // Ein Hund in einer Tier-Variable!
        tiere[1] = minka;    // Eine Katze in einer Tier-Variable!
        tiere[2] = emma;     // Eine Kuh in einer Tier-Variable!
        
        // Alle Tiere machen Geräusche - hier passiert Polymorphismus!
        System.out.println("Alle Tiere machen Geräusche:");
        for (Tier tier : tiere) {
            tier.machGeräusch();  // Jedes Tier macht sein eigenes Geräusch!
        }
    }
}

/* Ausgabe:
Normale Aufrufe:
Bello bellt: Wau wau!
Minka miaut: Miau!
Emma muht: Muuuuh!

Mit Polymorphismus:
Alle Tiere machen Geräusche:
Bello bellt: Wau wau!
Minka miaut: Miau!
Emma muht: Muuuuh!
*/

Erklärung für Anfänger:

In diesem Beispiel passiert Folgendes:

  1. Wir erstellen verschiedene Tiere (Hund, Katze, Kuh)
  2. Dann speichern wir sie in einem Tier[]-Array - obwohl sie eigentlich unterschiedliche Klassen sind!
  3. Wenn wir machGeräusch() aufrufen, "weiß" Java trotzdem, welches Tier es ist, und ruft die richtige Methode auf

Dies ist die Magie des Polymorphismus: Obwohl wir nur mit Tier-Variablen arbeiten, hören wir von jedem Tier sein eigenes, spezielles Geräusch!

Aufgabe 3: Osterhasen-Helfer

Der Osterhase braucht Hilfe beim Osterfest! Erstelle ein Programm mit verschiedenen Osterhelfern.

  1. Schreibe eine Klasse Osterhelfer mit:
    • Eigenschaft: name (String)
    • Methode: helfen() - gibt aus, wie der Helfer beim Osterfest hilft
  2. Erstelle zwei oder drei Helfertypen, die von Osterhelfer erben, zum Beispiel:
    • Osterhase - versteckt Eier
    • Osterkueken - bemalt Eier
    • Erfinde einen eigenen Helfer!
  3. Jeder Helfertyp soll die helfen()-Methode überschreiben
  4. Erstelle eine Osterfest-Klasse, die viele Helfer in einem Array speichert
  5. Zeige, wie alle Helfer gemeinsam arbeiten, indem du für jeden die helfen()-Methode aufrufst
class Osterhelfer {
    String name;
    
    Osterhelfer(String name) {
        this.name = name;
    }
    
    void helfen() {
        System.out.println(name + " hilft irgendwie beim Osterfest.");
    }
}

class Osterhase extends Osterhelfer {
    Osterhase(String name) {
        super(name);  // Ruft den Konstruktor der Elternklasse auf
    }
    
    @Override
    void helfen() {
        System.out.println(name + " versteckt bunte Ostereier im Garten.");
    }
}

class Osterkueken extends Osterhelfer {
    Osterkueken(String name) {
        super(name);
    }
    
    @Override
    void helfen() {
        System.out.println(name + " bemalt Eier in schönen Farben.");
    }
}

class Osterbäcker extends Osterhelfer {
    Osterbäcker(String name) {
        super(name);
    }
    
    @Override
    void helfen() {
        System.out.println(name + " bäckt leckere Osterhasen aus Teig.");
    }
}

class Osterfest {
    private Osterhelfer[] helfer;
    
    Osterfest(Osterhelfer[] helfer) {
        this.helfer = helfer;
    }
    
    void feiern() {
        System.out.println("=== Das Osterfest beginnt! ===");
        System.out.println("Alle Helfer sind fleißig bei der Arbeit:");
        
        // Jeder Helfer hilft auf seine eigene Art
        for (Osterhelfer h : helfer) {
            h.helfen();  // Hier passiert Polymorphismus!
        }
        
        System.out.println("=== Das Osterfest war ein voller Erfolg! ===");
    }
}

public class OsterDemo {
    public static void main(String[] args) {
        // Verschiedene Osterhelfer erstellen
        Osterhelfer[] alleHelfer = new Osterhelfer[4];
        alleHelfer[0] = new Osterhase("Hoppel");
        alleHelfer[1] = new Osterkueken("Piepsi");
        alleHelfer[2] = new Osterbäcker("Meister Keks");
        alleHelfer[3] = new Osterhelfer("Oma Inge");  // Ein normaler Helfer
        
        // Osterfest erstellen und feiern
        Osterfest fest = new Osterfest(alleHelfer);
        fest.feiern();
    }
}

/* Ausgabe:
=== Das Osterfest beginnt! ===
Alle Helfer sind fleißig bei der Arbeit:
Hoppel versteckt bunte Ostereier im Garten.
Piepsi bemalt Eier in schönen Farben.
Meister Keks bäckt leckere Osterhasen aus Teig.
Oma Inge hilft irgendwie beim Osterfest.
=== Das Osterfest war ein voller Erfolg! ===
*/

Erklärung:

Dieses Beispiel zeigt, wie Polymorphismus uns hilft, ein flexibles Programm zu schreiben:

  1. Die Osterfest-Klasse arbeitet nur mit Osterhelfer-Objekten
  2. Sie muss nicht wissen, welche Art von Helfern genau dabei sind
  3. Jeder Helfer weiß selbst, wie er helfen kann
  4. Wir können jederzeit neue Helfertypen hinzufügen, ohne den Code für das Osterfest zu ändern!

Das ist der große Vorteil von Polymorphismus: Wir können flexibel neue Unterklassen hinzufügen, ohne bestehenden Code anzupassen.

Zusammenfassung: Polymorphismus und Vererbung

  • Vererbung: Eine Klasse kann Eigenschaften und Methoden von einer anderen erben
  • Methoden überschreiben: Eine Kindklasse kann eine Methode der Elternklasse neu definieren
  • Polymorphismus: Objekte unterschiedlicher Klassen können auf den gleichen Methodenaufruf unterschiedlich reagieren

Die 3 Schritte zum Polymorphismus:

  1. Eine Elternklasse mit Methoden erstellen
  2. Kindklassen erstellen, die Methoden überschreiben
  3. Elternklassen-Variablen oder -Arrays verwenden, um Kindklassen-Objekte zu speichern

Einfache Erklärung zum Mitnehmen:

Stell dir vor, du hast eine Fernbedienung mit einem "Ein/Aus"-Knopf:

  • Bei einem Fernseher schaltet der Knopf den Bildschirm ein oder aus
  • Bei einer Stereoanlage schaltet er die Musik ein oder aus
  • Bei einer Lampe schaltet er das Licht ein oder aus

Der gleiche Knopf (die gleiche Methode) tut bei verschiedenen Geräten (Objekten) unterschiedliche Dinge - das ist Polymorphismus!
